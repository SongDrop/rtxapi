Hybrid Automated Dev VM + Collaborative Remote Editing Dashboard

This dashboard outlines how to spin up UE3 projects with a single VM image, manage multiple projects, and automate collaboration.

  • Use a base UE3 VM image for all projects.
  • On VM startup, a script can:
    • Connect to OpenVPN automatically
    • Check for each Unreal project repo (Forgejo/Git)
    • If missing ‚Üí git clone <repo>
    • If existing ‚Üí git pull to update
    • Map folders to UE3 or open automatically
  • This makes the VM project-agnostic: spin up multiple projects instantly without reinstalling everything.
  • VS2022 maintains a local copy of your project.
  • Browser-based IDEs like VS Code Online or Codespaces allow multiple collaborators to edit and push commits.
  • VM startup scripts can automatically pull new commits into VS2022.
  • Optionally, use webhooks or watcher scripts to trigger builds after code updates.
  • OpenVPN connects automatically on VM boot.
  • Incredibuild sees all agents and distributes builds automatically.
  • No manual intervention required for network or build setup.
  • Edit locally in VS2022 or via VS Code Online.
  • VM triggers headless builds via PowerShell + MSBuild or UnrealBuildTool.
  • Build logs captured and optional notifications via webhook.
  • Fully automated; no human clicks required.
  • Multiple VM profiles for different projects.
  • Auto-update Unreal, VS, and dependencies without manual intervention.
  • Shortcut to instantly open latest synced project.
  • Automated rollback to previous snapshot if build fails.
  1. Spin up UE3 Project 1 using base VM + git repo.
  2. Inspect files, test, take notes.
  3. Delete the instance after finishing.
  4. Spin up UE3 Project 2 with the same base VM + git repo.
  5. Repeat for all projects while keeping the base VM image unchanged.
