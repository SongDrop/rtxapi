How It Works

1

Provide Your Credentials

You give us your GitHub repository details and Azure credentials. We handle the rest.

2

Automatic Setup

Our system creates your Azure Function App, sets up the GitHub Actions workflow, and configures all environment variables.

// Example of the automated workflow creation name: Build and deploy Python project to Azure Function App jobs: build: runs-on: ubuntu-latest steps: - name: Checkout repository uses: actions/checkout@v4 - name: Setup Python uses: actions/setup-python@v5 - name: Install dependencies run: pip install -r requirements.txt
3

Template Sync

Your repository is automatically synced with our battle-tested template, giving you all the pre-configured applications.

4

Ready to Use

Your complete platform is ready with all applications accessible via subdomains and everything configured for production use.

Extending The Platform

The real power comes from easily adding your own applications. Here's how it works:

// Simple structure for adding new services { "logo": "https://example.com/logo.png", "title": "Your Custom App", "description": "Description of your amazing application", "github_url": "https://github.com/yourusername/your-app", "subdomain": "yourapp", "username": "admin", "password": "yourpassword123!", "azure_create_api_url": "/api/create_vm_yourapp", }

API Endpoints

Each application has its own API endpoint following the pattern:

// Windows VM creation endpoint /api/create_vm_windows // Ubuntu VM creation endpoint /api/create_vm_ubuntu // Your custom application endpoint /api/create_vm_yourapp

Customization Files

Each service has three key files for complete customization:

__init__.py

VM creation logic - handles Azure resource provisioning

generate_setup.py

Software installation scripts - your application setup

send_html_email.py

Email notifications - customized communication templates

Getting Started

1

Prerequisites

Before you begin, make sure you have:

  • A GitHub repository (format: yourusername/repo-name)
  • A GitHub Personal Access Token with repo permissions
  • Your Azure subscription details
  • A domain name for your services
2

Installation Process

Our system will automatically:

  • Create your complete API infrastructure
  • Set up all development tools (Forgejo, VSCode, Plane, etc.)
  • Configure automated deployment pipelines
  • Set up email service and monitoring
  • Sync with our template for continuous updates
3

Adding Custom Applications

Once setup is complete, you can easily add your own applications by:

  1. Creating a new JSON entry in your applications list
  2. Implementing the three key files (__init__.py, generate_setup.py, send_html_email.py)
  3. Your new application will automatically be integrated into the platform
