DISPOSABLE DEVELOPMENT ENVIRONMENT

Spin up fully-prepared, disposable developer workstations in the cloud — automated, collaborative, and ready-to-use.

Overview

This system turns a VM into a disposable, fully-prepared development environment that is automated and collaborative. Spin up multiple Unreal Engine projects, sync code, connect to OpenVPN, and trigger builds without manual intervention. Each VM is project-agnostic — clone, test, and delete instances efficiently.

Why "Disposable" matters

  • Massive reduction in upfront hardware costs — no need to buy expensive workstations for every team member.
  • Teams run on lightweight devices (laptops, tablets) while heavy builds execute in cloud VMs.
  • Easy reproducibility and sharing: one golden base image can serve many projects.

Primary components

  • Image builder: Create and snapshot Windows VMs; convert to Hyper-V fixed-size VHDs.
  • Automation API: Provision, snapshot, convert, export and manage VMs programmatically.
  • Control center: One-stop for deploying pre-configured apps and developer tooling.
  • VHDUSB: An Electron tool to download VHD to USB for offline/portable use.

Connect with Azure

The platform uses Azure to run Windows and Ubuntu VMs on demand. Key steps to make it work:

  1. Create an Azure Function App (Flex Consumption recommended for cost-efficiency).
  2. Enable GitHub deployment or set up deployment via publish profile / service principal.
  3. Provision required quotas (vCPUs, disk storage, GPU if needed) and set up networking.
  4. Store secrets (publish profile, Azure credentials, GitHub token) securely in your repo or Azure Key Vault.

Automation notes

  • A dedicated GitHub Actions workflow builds and packages the Python function/API, installs dependencies, and deploys to Azure.
  • Use app settings (environment variables) to inject configuration like OPENAI_API_KEY, storage SAS tokens, etc.
  • Prefer CLI/ARM/Bicep or Terraform to provision resources reproducibly for production deployments.

Control Panel — pre-configured apps

The Control Panel is your personal mini-portal: manage Windows/Ubuntu VMs, stream desktops, and access pre-installed services.

Core apps included

  • Win10 Remote — GPU-enabled Windows machines for heavy workloads.
  • Forgejo — self-hosted Git service.
  • VSCode Server — browser-based editor for quick edits.
  • MailCow, Plane, N8N — mail, project management and workflow automation.
  • ByteStash, Answer, Zammad — developer utilities and team tools.

How to use

  1. Pick an app from the Control Panel and click Create.
  2. Configuration templates (username/password, subdomain, repo) are filled from your JSON manifest.
  3. Service is deployed automatically; use the provided subdomain or custom domain.

API — automation backbone

The API is the central layer that makes disposable development environments scalable and reproducible.

The API gives https URL links to programmatically manage VMs, snapshots, image conversion and export. This is how automation & scale happen.

Once you installed your api, it will clone the original builds from our repo, and spins up a Premium Function App in your Microsoft Azure. This will be synced all the time with your private github repo(you need GitHub account, and a token), so you can access and modify and sync your own API even in VSCode online (VSCode Online - a code editor running in your browser - which can be setup in the Controller panel). This is important to be able to add or remove new one-click softwares when you wish.

Key endpoints

  • /api/create_vm_s_control — Creates the control center to use the service - Very important!
  • /api/create_vm_s_api — Setup the API for you to use - Very important!
  • /api/api_gateway — helps to add or remove new api functions without breaking the system. Also adds extra security.
  • /api/create_vm_windows — create a Windows virtual machine with configurable options such as size, disk type, and networking.
  • /api/create_vm_ubuntu — create an Linux Ubuntu-based virtual machine for browser based softwares

  • /api/export_vm — take a snapshot of a VM, generating a SAS URL, spin of a Windows 10 with Hyper-V, automatising fixed size bootable vhd creation, and upload it to storage and give a downloadable url to put onto usb. can help you to download 'fixed-size-bootable.vhd' from URL to USB pendrive
  • /api/list_vm — retrieve a list of all available virtual machines, including their details and statuses from the same group.

Custom Apps — extend the ecosystem

The platform lets you add or remove a custom service list of softwares of your choice dynamically via a JSON manifest and a small set of customization files. - In order to costumise you need to setup your own API in the previous section.

Service manifest example

{
  "logo":"https://example.com/logo.png",
  "title":"Your Custom App",
  "description":"What the app does",
  "github_url":"https://github.com/you/your-app",
  "subdomain":"yourapp",
  "username":"admin",
  "password":"strong-password",
  "azure_create_api_url":"/api/create_vm_yourapp"
}

Three files per service(using Azure Python API v1 - battle tested)

  • __init__.py — function logic to provision the VM or service.
  • generate_setup.py — scripts (PowerShell/Bash) installed into the VM to configure software. - This is where the softwares automatically installed - this is what you focus on to create
  • send_html_email.py — email templates/notifications on successful deploy or errors. - SMTP Email Server REQUIRED!
  • html_email.py — email generation with printf returning the HTML with your dynamic imputs.
  • function.json — make sure you always edit this for the correct name for your api function "route": "create_vm_s_", without this the api function you add won't be recognised by api_gateway

Drop the manifest + files into your repo and the platform will sync and expose the new service in the Control Panel.

Create — building images & workflows

This is the workflow to go from a fresh VM to a fully-configured, exportable image:

  1. create_vm_windows / create_vm_ubuntu — provision a VM from templates.
  2. Install software — automatically via Chocolatey/Winget or custom scripts (generate_setup).
  3. snapshot_vm — capture the configured state for rollback or cloning.
  4. create_vm_hyperv — prepare VM for Hyper-V conversion (enable Hyper-V, convert disks to fixed-size VHD).
  5. export_vm — it creates a downloadable fixed-size bootable VHD (SAS URL) what you can save to a portable USB - can help you to download vhd to usb

Automation tips

  • Use generate_setup.py to orchestrate runtime provisioning and post-reboot helpers (network quieting, shortcuts).
  • Package complex installers via Chocolatey or Winget where possible.
  • For interactive installers (legacy games or tools), use AutoIt or PowerShell automation scripts to replicate UI steps headlessly.

VHDUSB — Export & offline usage

The VHDUSB workflow is what lets you take a cloud VM and make it physically portable on a USB stick:

How it works

  1. Create and snapshot VM in Azure (full configured system).
  2. Convert disks to a fixed-size Hyper-V VHD (bootable) using the Hyper-V tooling and internal scripts.
  3. Upload the VHD to Azure Storage and generate a SAS download URL (export_vm provides this).
  4. Use vhdusbdownloader (Electron app) to download the VHD directly to a USB drive and make it bootable.

Notes & caveats

  • Creating very large OS VHDs (eg. 256GB) via the portal is slow/manual; automation saves hours.
  • Ensure the target USB drive is fast and large enough; fixed-size VHDs require contiguous space.
  • Test bootability on target hardware — differences in firmware (UEFI/legacy) matter.

Hybrid Automated Dev VM + Collaborative Remote Editing

This workflow combines automated VM provisioning with real-time collaboration tools — giving you one disposable image for multiple Unreal Engine projects, synced automatically with Git and IDEs.

1️⃣ VM Setup for Multiple UE Projects

  • Base UE3 VM image reused across projects.
  • On startup: PowerShell script connects to OpenVPN, checks repos, clones/pulls updates, and auto-opens in UE3.
  • Result: project-agnostic, instant spin-up with zero reinstall.

2️⃣ Git Integration + Collaboration

  • VS2022 runs locally with a synced copy of your project.
  • VS Code Online / Codespaces allows browser-based editing.
  • Commits pulled automatically into VS2022; optional webhooks trigger rebuilds.

3️⃣ OpenVPN + Incredibuild Automation

  • OpenVPN starts automatically on VM boot.
  • Incredibuild detects all agents and distributes builds with no manual setup.

4️⃣ Compilation Workflow

  • Edit locally in VS2022 or remotely in VS Code Online.
  • Commits trigger headless builds via MSBuild or UnrealBuildTool.
  • Logs captured; optional webhooks notify teams.
  • No human clicks — fully automated.

5️⃣ Optional Enhancements

  • VM profiles per project.
  • Auto-updates for Unreal, VS, and dependencies.
  • Shortcuts to open latest project instantly.
  • Rollback to snapshot if builds fail.

BoxArt — packaging your game or USB

BoxArt is the visual design for your game DVD, USB development kit, or software package. It helps make your project look professional and appealing.

Design tips

  • Create a front cover, spine, and back layout (for DVD cases or USB sleeves).
  • Include logos, screenshots, and key information about your software or game.
  • Use design tools like Photoshop, GIMP, Canva, or Figma for high-quality results.
  • Keep dimensions consistent with your medium: DVD case, USB sleeve, or digital thumbnail.
  • Consider adding a QR code or download link for users to quickly access the content.
  • Maintain branding and style consistency with your other materials.

Release — shipping your game or app

Once builds are automated and reproducible, releasing becomes a matter of packaging and distribution.

Formats

  • Cloud image (VHD / VM image) for enterprise or partner distribution.
  • Portable USB image (via VHDUSB) for offline demos, events, or QA.
  • Traditional game packages (installers, ZIPs, Steam/Epic builds) produced from the automated build pipelines.

Distribution strategies

  • Continuous Delivery: auto-upload build artifacts to storage/CDN and notify QA via webhooks.
  • Staged releases: snapshot stable VMs and export them as golden images for distribution to partners.
  • Local testing: give QA a USB with the VHD for reproducible offline testing environments.
