How to Build & Deploy AudioScript™️

Frontend

AudioScript™️ has two main HTML pages:

  • index.html → User-facing frontend.
  • master.html → Admin panel for managing characters, moods, and recordings.

How AudioScript™️ Works Like Bitbucket for Voice Recording

1️⃣ Users & Authentication

Users log in via index.html with a username and password.

Login verifies credentials via PostgreSQL (local or remote).

If authenticated, the system loads their assigned project.

2️⃣ Voice Line Versioning (Instead of Code Commits)

Users create "commits" by adding/modifying:

  • subtitles_action → What's being said.
  • subtitles_reaction → Response to an event.
  • a_character → Which character speaks.
  • mood → Emotion in the voice line.
  • language → Voice line language.
  • [takes] → Multiple recorded versions.
  • linenumber → Specific script line.

These are stored similarly to Git commits, allowing tracking and version control.

3️⃣ Master Controls (master.html)

The admin panel allows:

  • Setting permissions (who can edit/add/remove).
  • Approving/rejecting "pull requests" (submitted voice lines).
  • Managing branches (e.g., different versions of a script).

4️⃣ Git LFS for Audio Storage

Audio files (.wav, .mp3, etc.) are stored using Git LFS instead of regular Git tracking.

PostgreSQL stores metadata (who recorded it, versioning, timestamps).

Backend Setup

Bitbucket

Store your project and enable Git LFS for audio files:

git lfs install
    git lfs track "*.wav"
    git add .gitattributes

PostgreSQL

Install and configure PostgreSQL:

sudo apt update
    sudo apt install postgresql postgresql-contrib
    sudo -i -u postgres
    psql

Create a database and user:

CREATE DATABASE audioscript;
    CREATE USER audioscript_user WITH ENCRYPTED PASSWORD 'yourpassword';
    GRANT ALL PRIVILEGES ON DATABASE audioscript TO audioscript_user;

Docker Setup

Create a Dockerfile to containerize your app:

FROM ubuntu:latest
    RUN apt update && apt install -y postgresql git-lfs
    COPY . /app
    WORKDIR /app
    CMD ["./AudioScript™️.exe"]

Running Locally or in the Cloud

Local Setup

docker build -t audioscript .
    docker run -p 8080:8080 audioscript

Cloud Deployment

Use a cloud provider (AWS, GCP, or Azure) and assign a static IP to your instance.

Accessing the App

  • Local: http://localhost:8080/index.html
  • Admin: http://localhost:8080/master.html
  • Cloud: http://your-cloud-ip/index.html
