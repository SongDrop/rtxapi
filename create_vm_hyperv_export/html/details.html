What you do

High-level flow
1. Spin up the VM
Create the NV6ads_A10_v5 instance, attach a 256 GB OS disk (managed disk / Premium recommended).
2. Build & pre-install
Install softwares on Windows 10, prepare the “clonable workstation” snapshot (example: 10 hours for first-time setup).
3. Convert & compact image
Use the dynamic->compact->fixed pipeline to produce an Azure-ready fixed VHD image.
4. Export or clone
Export image for reuse, or create images for remote clones (costs shown below).
5. Auto-cleanup
VM is deleted (ephemeral), disks removed as needed to avoid ongoing compute charges.
6. Download uploaded vhd
Use vhdusbdownloader to download the `your-os-fixed-size-bootable.vhd` to put it on usb pendrive.
7. Create boxart for your usb
Follow this tutorial to learn how to build your first box for yur development environment built in th cloud.
8. Make your game
Once you setup your development environment remotely and locally,the production can begun, which is the fun part. Use team development tools to fasten your build times, and ship a product in 6 months.
9. Design your end-user product box
Nw you have your own development environment box, and you finished your game, it time to design the box-arts for your physical distribution.
10. Release digitally and physically your game
Every game developers dream is to reach this point, when their video game can lands in the shops.

Export costs & options

Action Cost driver

How to use this system

  1. Launch NV6ads_A10_v5 with a 256 GB managed OS disk.
  2. Install and configure the workstation (your build, caches, etc.).
  3. Run the dynamic→compact→fixed conversion pipeline to create an Azure-ready VHD.
  4. Store the fixed VHD in Blob / managed disk (first-month cost shown above).
  5. Optionally export / clone for remote spin-ups — those clones are lightweight to instantiate but pay storage & egress when moved.
